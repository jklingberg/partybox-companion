# REST API — v1

Base path: `/api/v1`

Interactive docs (when daemon is running): `http://<host>:<port>/api/docs`

Two deployment shapes share this API:

- **Standalone daemon** (`partyboxd`) — serves the [daemon endpoints](#daemon-endpoints-partyboxd) on `127.0.0.1:8765` by default.
- **Appliance** (`partybox-companion`) — same daemon endpoints plus the [appliance endpoints](#appliance-endpoints-companion), served on port 80: `http://partybox.local/api/v1/...`

Examples below use the standalone daemon address; on the appliance, drop the port.

---

## Authentication

When an API key is configured, all endpoints except `GET /health` require an
`X-Api-Key` header.

```
X-Api-Key: <your-key>
```

Configure the key via environment variable:

```bash
PARTYBOXD_API__API_KEY=mysecretkey partyboxd
```

When `PARTYBOXD_API__API_KEY` is not set, authentication is disabled and all
requests are accepted. Suitable for trusted local networks.

**WebSocket** clients supply the key as a query parameter instead (browsers
cannot set custom headers on WebSocket connections):

```
ws://host:port/api/v1/events?api_key=<key>
```

---

## Error responses

All error responses share a consistent shape:

```json
{
  "detail": {
    "error": "<error_code>",
    "message": "<human-readable description>"
  }
}
```

| `error` | HTTP status | Meaning |
|---------|-------------|---------|
| `unauthorized` | 401 | Missing or invalid API key |
| `speaker_disconnected` | 503 | Speaker is not currently connected |
| `capability_unavailable` | 404 | Feature not supported by this speaker |

## Daemon endpoints (partyboxd)

Present in both deployment shapes.

### `GET /api/v1/health`

Daemon liveness check. Always returns **200**. No authentication required —
safe to poll from monitoring tools.

**Response:**

```json
{
  "status": "ok",
  "version": "0.1.0",
  "ble_connected": true,
  "audio_ready": true
}
```

| Field | Type | Description |
|-------|------|-------------|
| `status` | `string` | Always `"ok"` while the daemon is running |
| `version` | `string` | partyboxd version |
| `ble_connected` | `boolean` | Whether the daemon has an active BLE GATT connection to the speaker |
| `audio_ready` | `boolean\|null` | Whether the appliance can currently produce audio (A2DP connected). `null` when running standalone partyboxd without Companion |

**curl:**

```bash
curl http://partybox.local:8765/api/v1/health
```

**Python:**

```python
import httpx
r = httpx.get("http://partybox.local:8765/api/v1/health")
print(r.json())
# {'status': 'ok', 'version': '0.1.0', 'ble_connected': True, 'audio_ready': None}
```

### `GET /api/v1/speaker`

Current speaker connection state, firmware version, and battery level.

Always returns **200**. When the speaker is disconnected, `connected` is
`false` and all other fields are `null`.

**Response:**

```json
{
  "connected": true,
  "address": "AA:BB:CC:DD:EE:FF",
  "firmware": "26.2.10",
  "battery": null
}
```

| Field | Type | Description |
|-------|------|-------------|
| `connected` | `boolean` | Whether the daemon has an active BLE connection |
| `address` | `string\|null` | BLE address (informational — rotates, not stable) |
| `firmware` | `string\|null` | Firmware version, e.g. `"26.2.10"`. `null` if not yet read |
| `battery` | `integer\|null` | Battery percentage 0–100. `null` for mains-powered models |

**curl:**

```bash
curl -H "X-Api-Key: $KEY" http://partybox.local:8765/api/v1/speaker
```

**Python:**

```python
import httpx
r = httpx.get(
    "http://partybox.local:8765/api/v1/speaker",
    headers={"X-Api-Key": "mysecretkey"},
)
print(r.json())
# {'connected': True, 'address': 'AA:BB:CC:DD:EE:FF', 'firmware': '26.2.10', 'battery': None}
```

---

### `GET /api/v1/battery`

Current battery level as a percentage (0–100).

Only meaningful on battery-powered PartyBox models. Mains-powered models
(e.g. PartyBox 520) return **404**.

**Response — 200:**

```json
{
  "level": 84
}
```

**Responses:**

| Code | When |
|------|------|
| 200 | Battery level returned |
| 404 | This speaker does not have a battery |
| 503 | Speaker is not connected |

**curl:**

```bash
curl -H "X-Api-Key: $KEY" http://partybox.local:8765/api/v1/battery
```

**Python:**

```python
import httpx
r = httpx.get(
    "http://partybox.local:8765/api/v1/battery",
    headers={"X-Api-Key": "mysecretkey"},
)
if r.status_code == 200:
    print(r.json()["level"], "%")
elif r.status_code == 404:
    print("mains-powered speaker, no battery")
```

### `POST /api/v1/power/on`

Send a power-on command to the speaker.

Returns **204 No Content** on success. The response body is empty.

**Responses:**

| Code | When |
|------|------|
| 204 | Command accepted by the speaker |
| 503 | Speaker is not connected |

**curl:**

```bash
curl -X POST -H "X-Api-Key: $KEY" http://partybox.local:8765/api/v1/power/on
```

**Python:**

```python
import httpx
r = httpx.post(
    "http://partybox.local:8765/api/v1/power/on",
    headers={"X-Api-Key": "mysecretkey"},
)
r.raise_for_status()
```

### `WS /api/v1/events`

Real-time device event stream over WebSocket.

Connect with:

```
ws://partybox.local:8765/api/v1/events
ws://partybox.local:8765/api/v1/events?api_key=<key>   # with auth
```

The connection closes with code **4001** if the API key is invalid.

**Events** are JSON objects with a `type` discriminator field:

#### `connected`

Emitted when the daemon establishes a BLE connection with the speaker.

```json
{
  "type": "connected",
  "address": "AA:BB:CC:DD:EE:FF",
  "firmware": "26.2.10",
  "battery": null
}
```

#### `disconnected`

Emitted when the BLE connection is lost or terminated.

```json
{
  "type": "disconnected"
}
```

#### `power_changed`

Emitted after a successful `POST /api/v1/power/on` or `/power/off` command.

```json
{
  "type": "power_changed",
  "state": "on"
}
```

```json
{
  "type": "power_changed",
  "state": "off"
}
```

#### `ping`

Sent by the server every ~20 seconds when no other events occur. Clients may
use this to detect stale connections.

```json
{
  "type": "ping"
}
```

> A `volume_changed` event (`{"type": "volume_changed", "percent": 0-100}`) is
> defined for hardware volume notifications but is **not emitted yet** — the
> BLE volume opcode is unconfirmed. See [ADR-022](../adr/022-volume-authority.md).

**Python example:**

```python
import asyncio
import json
import websockets

async def watch():
    uri = "ws://partybox.local:8765/api/v1/events?api_key=mysecretkey"
    async with websockets.connect(uri) as ws:
        async for message in ws:
            event = json.loads(message)
            print(event)

asyncio.run(watch())
```

**JavaScript example:**

```javascript
const ws = new WebSocket("ws://partybox.local:8765/api/v1/events?api_key=mysecretkey");

ws.onmessage = (e) => {
  const event = JSON.parse(e.data);
  if (event.type === "connected") {
    console.log("Speaker connected, firmware:", event.firmware);
  } else if (event.type === "disconnected") {
    console.log("Speaker disconnected");
  } else if (event.type === "power_changed") {
    console.log("Power is now:", event.state);
  }
};
```

## Appliance endpoints (companion)

Only present when running `partybox-companion` (not standalone `partyboxd`).

These endpoints are **intentionally unauthenticated**: they expose read-only
appliance state or settings that require local network access anyway, and the
WiFi provisioning flow runs before any API key could be configured.

### `GET /api/v1/volume`

Current logical speaker volume (0–100). Tries the speaker hardware (BLE)
first; falls back to the last known software volume when the speaker is
disconnected or the BLE volume opcode is not yet confirmed. See
[ADR-022](../adr/022-volume-authority.md).

**Response:**

```json
{
  "level": 60,
  "source": "api"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `level` | `integer\|null` | Volume percentage 0–100. `null` if unknown |
| `source` | `string\|null` | Where the value came from: `ble`, `spotify`, `airplay`, or `api` |

### `POST /api/v1/volume`

Set the logical speaker volume.

**Request body:**

```json
{
  "level": 60
}
```

| Code | When |
|------|------|
| 204 | Volume accepted |
| 422 | `level` out of range or wrong type |

### `GET /api/v1/audio`

State of the Bluetooth A2DP audio connection. Always returns **200**.

**Response:**

```json
{
  "connected": true,
  "address": "AA:BB:CC:DD:EE:FF",
  "pairing_state": "idle"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `connected` | `boolean` | Whether the Pi has an active A2DP link to the speaker |
| `address` | `string\|null` | Classic Bluetooth MAC. `null` when no pairing has been performed |
| `pairing_state` | `string` | `idle`, `scanning`, `pairing`, or `failed` |

### `POST /api/v1/audio/pair`

Start a Bluetooth Classic pairing scan. Put the speaker into Bluetooth
pairing mode **before** calling this. Returns immediately; poll
`GET /api/v1/audio` to track progress (the scan runs for up to 60 s).

| Code | When |
|------|------|
| 202 | Pairing scan started |
| 409 | A pairing attempt is already in progress |
| 503 | Pairing service unavailable |

### `GET /api/v1/spotify`

State of the Spotify Connect service. Always returns **200**.

**Response:**

```json
{
  "running": true,
  "active": false,
  "device_name": "PartyBox Companion"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `running` | `boolean` | Whether librespot is running |
| `active` | `boolean` | Whether Spotify playback is currently in progress |
| `device_name` | `string` | Name shown in the Spotify Connect device list |

### `POST /api/v1/spotify/restart`

Restart the Spotify Connect service with the current saved configuration.
Call after `PUT /api/v1/config` to apply new Spotify settings without
rebooting. Returns **204**.

### `GET /api/v1/config`

Current appliance configuration. Always returns **200**; defaults are
returned on first boot.

**Response:**

```json
{
  "device_name": "PartyBox",
  "spotify_connect_name": "PartyBox Companion",
  "spotify_bitrate": 320,
  "audio_sink_address": "AA:BB:CC:DD:EE:FF"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `device_name` | `string` | Appliance display name |
| `spotify_connect_name` | `string` | Name advertised to Spotify Connect |
| `spotify_bitrate` | `integer` | `96`, `160`, or `320` (kbit/s) |
| `audio_sink_address` | `string\|null` | Paired speaker's Classic Bluetooth MAC |

### `PUT /api/v1/config`

Persist the appliance configuration. Body and response use the same shape as
`GET /api/v1/config`. Returns **200** with the saved configuration. To apply
changed Spotify settings, follow with `POST /api/v1/spotify/restart`.

### `GET /api/v1/debug/bundle`

Download a ZIP archive with diagnostic information: appliance version,
configuration, service status, system information, and recent log lines.
No sensitive data (API keys, credentials) is included.

Returns **200** with `Content-Type: application/zip`
(`partybox-debug-<timestamp>.zip`).

### `GET /api/v1/wifi/status`

WiFi provisioning state.

**Response:**

```json
{
  "state": "connected",
  "ap_ip": null,
  "reason": null,
  "message": null
}
```

| Field | Type | Description |
|-------|------|-------------|
| `state` | `string` | `unprovisioned`, `ap_active`, `connecting`, or `connected` |
| `ap_ip` | `string\|null` | IP of the appliance's own access point while provisioning |
| `reason` | `string\|null` | On failure: `authentication_failed`, `timeout`, `not_found`, or `unknown` |
| `message` | `string\|null` | Human-readable status suitable for a UI |

### `GET /api/v1/wifi/networks`

Nearby WiFi networks, sorted by signal strength.

**Response:**

```json
{
  "networks": [
    {"ssid": "HomeNet", "signal": 87, "security": "WPA2"}
  ]
}
```

### `POST /api/v1/wifi/connect`

Submit an SSID and optional password to join a WiFi network. Returns **204**
immediately; poll `GET /api/v1/wifi/status` to track progress. On failure the
appliance restores its access point and sets `reason`/`message` on the status
endpoint.

**Request body:**

```json
{
  "ssid": "HomeNet",
  "password": "secret"
}
```

## Home Assistant integration example

```yaml
# configuration.yaml
rest:
  - resource: http://partybox.local:8765/api/v1/speaker
    headers:
      X-Api-Key: !secret partybox_api_key
    scan_interval: 30
    sensor:
      - name: PartyBox Connected
        value_template: "{{ value_json.connected }}"
      - name: PartyBox Firmware
        value_template: "{{ value_json.firmware }}"

rest_command:
  partybox_power_on:
    url: http://partybox.local:8765/api/v1/power/on
    method: POST
    headers:
      X-Api-Key: !secret partybox_api_key

  partybox_power_off:
    url: http://partybox.local:8765/api/v1/power/off
    method: POST
    headers:
      X-Api-Key: !secret partybox_api_key
```
