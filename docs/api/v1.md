# REST API — v1

Base path: `/api/v1`

Interactive docs (when daemon is running): `http://<host>:<port>/api/docs`

---

## Authentication

When an API key is configured, all endpoints except `GET /health` require an
`X-Api-Key` header.

```
X-Api-Key: <your-key>
```

Configure the key via environment variable:

```bash
PARTYBOXD_API__API_KEY=mysecretkey partyboxd
```

When `PARTYBOXD_API__API_KEY` is not set, authentication is disabled and all
requests are accepted. Suitable for trusted local networks.

**WebSocket** clients supply the key as a query parameter instead (browsers
cannot set custom headers on WebSocket connections):

```
ws://host:port/api/v1/events?api_key=<key>
```

---

## Error responses

All error responses share a consistent shape:

```json
{
  "detail": {
    "error": "<error_code>",
    "message": "<human-readable description>"
  }
}
```

| `error` | HTTP status | Meaning |
|---------|-------------|---------|
| `unauthorized` | 401 | Missing or invalid API key |
| `speaker_disconnected` | 503 | Speaker is not currently connected |
| `capability_unavailable` | 404 | Feature not supported by this speaker |

## Endpoints

### `GET /api/v1/health`

Daemon liveness check. Always returns **200**. No authentication required.

**Response:**

```json
{
  "status": "ok",
  "speaker_connected": true
}
```

| Field | Type | Description |
|-------|------|-------------|
| `status` | `string` | Always `"ok"` while the daemon is running |
| `speaker_connected` | `boolean` | Whether the daemon has an active BLE connection |

**curl:**

```bash
curl http://partybox.local:8765/api/v1/health
```

**Python:**

```python
import httpx
r = httpx.get("http://partybox.local:8765/api/v1/health")
print(r.json())
# {'status': 'ok', 'speaker_connected': True}
```

---

### `GET /api/v1/speaker`

Current speaker connection state, firmware version, and battery level.

Always returns **200**. When the speaker is disconnected, `connected` is
`false` and all other fields are `null`.

**Response:**

```json
{
  "connected": true,
  "address": "AA:BB:CC:DD:EE:FF",
  "firmware": "26.2.10",
  "battery": null
}
```

| Field | Type | Description |
|-------|------|-------------|
| `connected` | `boolean` | Whether the daemon has an active BLE connection |
| `address` | `string\|null` | BLE address (informational — rotates, not stable) |
| `firmware` | `string\|null` | Firmware version, e.g. `"26.2.10"`. `null` if not yet read |
| `battery` | `integer\|null` | Battery percentage 0–100. `null` for mains-powered models |

**curl:**

```bash
curl -H "X-Api-Key: $KEY" http://partybox.local:8765/api/v1/speaker
```

**Python:**

```python
import httpx
r = httpx.get(
    "http://partybox.local:8765/api/v1/speaker",
    headers={"X-Api-Key": "mysecretkey"},
)
print(r.json())
# {'connected': True, 'address': 'AA:BB:CC:DD:EE:FF', 'firmware': '26.2.10', 'battery': None}
```

### `GET /api/v1/battery`

Current battery level as a percentage (0–100).

Only meaningful on battery-powered PartyBox models. Mains-powered models
(e.g. PartyBox 520) return **404**.

**Response — 200:**

```json
{
  "level": 84
}
```

**Responses:**

| Code | When |
|------|------|
| 200 | Battery level returned |
| 404 | This speaker does not have a battery |
| 503 | Speaker is not connected |

**curl:**

```bash
curl -H "X-Api-Key: $KEY" http://partybox.local:8765/api/v1/battery
```

**Python:**

```python
import httpx
r = httpx.get(
    "http://partybox.local:8765/api/v1/battery",
    headers={"X-Api-Key": "mysecretkey"},
)
if r.status_code == 200:
    print(r.json()["level"], "%")
elif r.status_code == 404:
    print("mains-powered speaker, no battery")
```

### `POST /api/v1/power/on`

Send a power-on command to the speaker.

Returns **204 No Content** on success. The response body is empty.

**Responses:**

| Code | When |
|------|------|
| 204 | Command accepted by the speaker |
| 503 | Speaker is not connected |

**curl:**

```bash
curl -X POST -H "X-Api-Key: $KEY" http://partybox.local:8765/api/v1/power/on
```

**Python:**

```python
import httpx
r = httpx.post(
    "http://partybox.local:8765/api/v1/power/on",
    headers={"X-Api-Key": "mysecretkey"},
)
r.raise_for_status()
```

### `WS /api/v1/events`

Real-time device event stream over WebSocket.

Connect with:

```
ws://partybox.local:8765/api/v1/events
ws://partybox.local:8765/api/v1/events?api_key=<key>   # with auth
```

The connection closes with code **4001** if the API key is invalid.

**Events** are JSON objects with a `type` discriminator field:

#### `connected`

Emitted when the daemon establishes a BLE connection with the speaker.

```json
{
  "type": "connected",
  "address": "AA:BB:CC:DD:EE:FF",
  "firmware": "26.2.10",
  "battery": null
}
```

#### `disconnected`

Emitted when the BLE connection is lost or terminated.

```json
{
  "type": "disconnected"
}
```

#### `power_changed`

Emitted after a successful `POST /api/v1/power/on` or `/power/off` command.

```json
{
  "type": "power_changed",
  "state": "on"
}
```

```json
{
  "type": "power_changed",
  "state": "off"
}
```

#### `ping`

Sent by the server every ~20 seconds when no other events occur. Clients may
use this to detect stale connections.

```json
{
  "type": "ping"
}
```

**Python example:**

```python
import asyncio
import json
import websockets

async def watch():
    uri = "ws://partybox.local:8765/api/v1/events?api_key=mysecretkey"
    async with websockets.connect(uri) as ws:
        async for message in ws:
            event = json.loads(message)
            print(event)

asyncio.run(watch())
```

**JavaScript example:**

```javascript
const ws = new WebSocket("ws://partybox.local:8765/api/v1/events?api_key=mysecretkey");

ws.onmessage = (e) => {
  const event = JSON.parse(e.data);
  if (event.type === "connected") {
    console.log("Speaker connected, firmware:", event.firmware);
  } else if (event.type === "disconnected") {
    console.log("Speaker disconnected");
  } else if (event.type === "power_changed") {
    console.log("Power is now:", event.state);
  }
};
```

## Home Assistant integration example

```yaml
# configuration.yaml
rest:
  - resource: http://partybox.local:8765/api/v1/speaker
    headers:
      X-Api-Key: !secret partybox_api_key
    scan_interval: 30
    sensor:
      - name: PartyBox Connected
        value_template: "{{ value_json.connected }}"
      - name: PartyBox Firmware
        value_template: "{{ value_json.firmware }}"

rest_command:
  partybox_power_on:
    url: http://partybox.local:8765/api/v1/power/on
    method: POST
    headers:
      X-Api-Key: !secret partybox_api_key

  partybox_power_off:
    url: http://partybox.local:8765/api/v1/power/off
    method: POST
    headers:
      X-Api-Key: !secret partybox_api_key
```
